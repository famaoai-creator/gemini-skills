# 要件定義書：Customer Portal Sync v2
**準拠**: IPA ユーザのための要件定義ガイド

## 1. 業務要求
- **目的**: 営業担当者が外出先から顧客情報を「1タップ」で確認・更新できる高信頼なポータルの構築。
- **デザイン方針 (`ux-auditor`)**: 認知負荷を最小限に抑える「シングルカラム・カードレイアウト」。モバイルファースト。

## 2. 機能要件 (TDD対象)
- **F1: 高速顧客検索**: 部分一致検索および大文字小文字を区別しない検索。
- **F2: 認証**: JWTを用いたステートレス認証。

## 3. 品質目標
- **テストカバレッジ**: 90% 以上。
- **セキュリティ**: 秘密情報のコード内混入ゼロ。
