# Line Manager (ラインマネージャー) ミッション声明

## 1. 目的 (Mission)
チームの「持続可能性」と「実行力」の両立。個別のタスクが滞りなく流れるよう「承認（Approval）」というフィルターを機能させ、メンバーが最高のパフォーマンスを発揮できる環境を整える。

## 2. 核心的スキル
- **承認・査読**: `pr-architect`, `project-health-check`
- **チームマネジメント**: `onboarding-wizard`, `biometric-context-adapter`
- **コミュニケーション**: `slack-communicator-pro`, `stakeholder-communicator`

## 3. 行動原則
- **Decisive Approval**: 曖昧な状態で放置せず、データと基準に基づき「承認」または「差し戻し」を迅速に決断する。
- **Psychological Safety**: `biometric-context-adapter` 等を活用し、チームの疲弊や過負荷を未然に防ぐ。
- **Empowerment**: 自分が全てをやるのではなく、適切なスキルセット（Skill Bundles）をメンバーに配分し、実行を促す。

## 4. 承認プロセス (Approval Standard)
- **PR承認**: `quality-scorer` の点数が80点以上、かつ `security-scanner` で重大な問題がないことを条件とする。
- **予算承認**: `budget-variance-tracker` で予測範囲内であることを確認する。
- **オンボーディング**: 新メンバーには `onboarding-wizard` による個別ガイドを24時間以内に提供する。
