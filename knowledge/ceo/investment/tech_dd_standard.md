# 技術デューデリジェンス (Tech DD) 標準基準

投資検討先（Startup）の技術的価値とリスクを評価するためのフレームワーク。コードアクセス権がない場合でも適用可能な「ブラックボックス評価」を含む。

## 1. アーキテクチャとスケーラビリティ
- **確認事項**:
    - 「100倍のトラフィック」に耐えうる設計か？（単一障害点の有無）
    - 技術的負債の返済計画はあるか？
- **ブラックボックス評価**:
    - 採用JDの必須スキル（枯れた技術か、流行りの技術か）から安定性を推測。
    - サービスダウンの頻度や速度低下の口コミ。

## 2. チームと開発プロセス
- **確認事項**:
    - **バス係数**: キーマン（CTO等）が抜けた瞬間に開発が止まらないか。
    - **自動化レベル**: CI/CD、テスト自動化への投資度合い。
- **ブラックボックス評価**:
    - エンジニアブログの更新頻度と内容の深さ（技術広報への熱量）。
    - 離職率やGlassdoor等の評価。

## 3. 知的財産 (IP) とコンプライアンス
- **確認事項**:
    - 競合特許の侵害リスク（FTO調査）。
    - OSSライセンスの管理体制。
    - コアアルゴリズムは自社開発か、APIラッパーか。

## 4. CTOインタビュー用「キラークエスチョン」
- 「現在のアーキテクチャで、最も後悔している選択は何ですか？」（誠実さと学習能力を見る）
- 「今後半年で、開発チームを倍にするならどこに配置しますか？」（ボトルネックの把握）
