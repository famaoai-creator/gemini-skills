# 定量的品質管理指標：バグ密度・テスト密度 (Quality Metrics)

IPA（情報処理推進機構）の統計データに基づいた、ウォーターフォール開発における品質評価の目安。

## 1. 基準値 (Benchmarks)
ソースコード 1,000行 (1 KLOC) あたりの指標。

| 指標 | 目安 (平均値) | 判定の考え方 |
| :--- | :--- | :--- |
| **テスト密度** | 10.0 ～ 30.0 ケース/KLOC | 少なすぎる場合は網羅性不足。多すぎる場合は効率性を確認。 |
| **バグ密度** | 2.0 ～ 5.0 件/KLOC | 極端に少ない場合はテスト不足を疑う。多い場合は設計不備。 |

## 2. 信頼性成長曲線 (Gompertz Curve)
- テスト実施数（累積）に対し、バグ累積発見数が「S字」を描いて収束していることを確認する。
- 収束していない（バグが出続けている）状態での工程完了は原則禁止。

## 3. 品質分析の黄金律
- **テスト密度 OK ＋ バグ密度 低**: 
    - 可能性1: 極めて高品質。
    - 可能性2: **テスト項目が単純すぎてバグを見逃している。** (再レビューが必要)
- **テスト密度 低 ＋ バグ密度 低**: 
    - **品質リスク大。** 未消化のバグが次工程で爆発する予兆。
- **バグ密度 極端に高**: 
    - 該当モジュールの再設計（作り直し）を検討。
