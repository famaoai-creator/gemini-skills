# 監視設定ベストプラクティス・チェックリスト

サービス稼働率（SLO）を維持するために、最低限設定すべき監視項目の基準。

## 1. 死活・リソース監視
- [ ] **プロセス死活**: サービスが応答を返しているか（Health Check）。
- [ ] **リソース枯渇**: CPU/Mem 80% 超過、Disk 90% 超過のアラート。
- [ ] **デッドライン**: Lambda の実行タイムアウトや API 応答遅延の監視。

## 2. アプリケーション監視
- [ ] **エラーレート**: 5xx エラーの急増を検知しているか。
- [ ] **ログ監視**: ERROR/FATAL ログのリアルタイム検知と Slack 通知。

## 3. 通知・運用統制
- [ ] **通知先**: 適切な担当者（またはオンコールツール）に紐付いているか。
- [ ] **自動復旧連携**: 監視アラートが `self-healing-orchestrator` のトリガーになっているか。
