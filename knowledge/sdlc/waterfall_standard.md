# ウォーターフォール開発標準：工程と成果物 (SDLC Standard)

本ドキュメントは、IPAの標準的な指針に基づき、ウォーターフォール型プロジェクトにおける各工程の定義と、必須・推奨される成果物をまとめたものである。

## 1. 開発工程定義

| 工程名               | 目的                                             | 主要な成果物                                             |
| :------------------- | :----------------------------------------------- | :------------------------------------------------------- |
| **要件定義**         | システムの機能・非機能要件を確定させる。         | 要件定義書、**レビュー指摘回答一覧**                     |
| **基本設計**         | インターフェースと全体構成を定義。               | 基本設計書、**レビュー指摘回答一覧**                     |
| **詳細設計**         | 物理構造を定義。                                 | 詳細設計書、**レビュー指摘回答一覧**                     |
| **実装**             | プログラムを構築。                               | ソースコード、**コードレビュー記録**                     |
| **テスト**           | 品質を検証。                                     | 各テスト計画/項目/報告書、**バグ管理表**                 |
| **結合テスト (IT)**  | モジュール間のインターフェースと機能連携を検証。 | 結合テスト計画書、結合テスト項目表、結合テスト実施報告書 |
| **総合テスト (ST)**  | システム全体の要件充足性と非機能を検証。         | 総合テスト計画書、総合テスト項目表、総合テスト実施報告書 |
| **受入テスト (UAT)** | 業務遂行が可能か、ユーザー視点で最終検証。       | 受入テスト計画書、受入完了報告書                         |

## 2. 工程完了の条件 (Exit Criteria)

- 各工程の成果物が作成され、担当者・承認者によるレビューが完了していること。
- 指摘事項（バグ・不備）がすべて修正され、完了または次工程への持ち越しが合意されていること。
