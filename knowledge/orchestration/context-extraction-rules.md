# コンテキスト抽出・要約ルール (Context Extraction Rules)

`token-economist` が情報を圧縮する際に、品質を維持するための優先基準。

## 1. 削除・圧縮対象 (Noise)
- **定型的挨拶・敬語**: 日本語の「お世話になっております」等の枕詞は全削除。
- **冗長な説明**: 文脈から明らかなボイラープレートコードや、一般的な用語解説。
- **重複情報**: 複数ファイルで共通するライセンスヘッダー等。

## 2. 絶対維持対象 (Signal)
- **固有名詞・数値**: プロジェクト名、ポート番号、金額、日付、APIエンドポイント。
- **依存関係**: バージョン番号、インポートパス、ライブラリ名。
- **意思決定の記録**: `intent-archivist` が特定した「なぜ」にあたる記述。

## 3. 要約の深さ
- **L1 (Discovery)**: タイトルと概要のみ (100文字以内)。
- **L2 (Technical)**: インターフェース定義と主要ロジックのみ。
- **L3 (Strategic)**: 結論とROI、およびリスク要因のみ。
