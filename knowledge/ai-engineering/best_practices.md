# AIエンジニアリング・ベストプラクティス

生成AIをシステムに統合する際の、品質・コスト・信頼性のための現代的基準。

## 1. 非機能要求 (AI-Specific NFR)

- **データ主権 (Data Residency)**: データの物理的な保存場所と法的境界線の明記。
- **Human-in-the-Loop**: AIの判断を「いつ、誰が、どのように」補正・承認するかという人間介入の境界定義。
- **AIトレーサビリティ**: 特定のアウトプットがどのモデル、どのプロンプト、どのデータから生成されたかの追跡。

## 2. セキュリティとプライバシー

- **自動スクラビング**: PII（個人情報）の検知・マスキングパイプライン。
- **プロンプトインジェクション対策**: 入力バリデーションとシステムプロンプトの堅牢化。

## 3. コスト・パフォーマンス

- **ハイブリッド・アーキテクチャ**: 軽量モデルとLLMの使い分け。
- **サンプリング分析**: コスト抑制のための統計的抽出分析。

## 4. モバイル・ネイティブ TDD

- **ロジックの分離**: UIから独立した Swift Package / Kotlin Library としてコアロジックを切り出し、シミュレータを使わず `swift test` 等で高速に TDD を回す。
- **実機エビデンス**: PR 時には、AI 生成物だけでなく、ローカル環境でのビルド・テスト実行ログを必須の「証拠」として添付する。
