# Sovereign Terminal Hosting Protocol

Web ブラウザを通じてエコシステムの PTY（疑似端末）へ直接アクセスするためのアーキテクチャ設計。

## 1. 概要
本プロトコルは、`node-pty` をバックエンド・エンジンとし、WebSocket (ws) をブリッジとして、フロントエンドの `xterm.js` と双方向・リアルタイムに通信する高忠実度なターミナル環境を定義する。

## 2. コア・コンポーネント

### 2.1 Backend: PTY Engine (`node-pty`)
- **役割**: ネイティブ・シェルの生成とプロセス管理。
- **重要事項 (macOS/Darwin)**:
    - 実行環境（Terminal, iTerm2, VSCode 等）に対して、**「ハードディスクへのフルアクセス」**権限を付与する必要がある。
    - 権限が不足している場合、`pty.spawn` は成功してもシェルの起動（`/bin/bash` 等）でハングまたは拒絶される。

### 2.2 Bridge: WebSocket Server (`ws`)
- **役割**: PTY のバイナリデータとブラウザ間の低遅延ブリッジ。
- **通信プロトコル**:
    - **JSON 形式**: `input`（キー入力データ）と `resize`（ウィンドウサイズ変更命令）を区別して受信する。
    - **Raw 形式**: PTY からの出力は、ANSI エスケープシーケンスを含む生の文字列としてブラウザへ転送される。

### 2.3 Frontend: Web Terminal (`xterm.js`)
- **役割**: 高速なターミナル・レンダリングとユーザー入力のキャプチャ。
- **拡張機能**:
    - `FitAddon`: ブラウザ・ウィンドウのリサイズに追従し、適切な `cols/rows` を計算する。
- **IME 回避戦略**:
    - `term.focus()` を初期化時およびクリック時に強制し、ブラウザのテキスト入力支援機能を抑制する。

## 3. セットアップと起動
```bash
# 依存関係のインストール
pnpm install node-pty ws express

# サーバーの起動
PORT=4321 node dist/presence/bridge/terminal/server.js
```

## 4. セキュリティとガバナンス
- **アクセス制御**: 本ターミナルは主権者の物理的なローカルマシン上での稼働を前提としている。
- **不揮発性**: セッションはブラウザの切断とともに終了（`rt.kill()`）される。
