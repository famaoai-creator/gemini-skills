# Project Management Plan (PMP) - 13 Standard Items

PMラボラトリーの知見に基づく、プロジェクト計画書の「完全網羅」標準。

## 1. 構成要素と論理的必然性 (13 Key Items)

| No | 項目 (Item) | 役割と必然性 |
| :--- | :--- | :--- |
| **1** | **プロジェクトの概要** | **何のためにやるのか(Why)**。全関係者の目線を合わせる北極星。 |
| **2** | **システム構成** | **何を作るのか(What)**。全体像を可視化し、インターフェース不整合を防ぐ。 |
| **3** | **体制** | **誰が責任を持つのか(Who)**。指揮系統を明確にし、意志決定の停滞を防ぐ。 |
| **4** | **マスタスケジュール** | **いつ終わるのか(When)**。主要マイルストーンを定義し、進捗のモノサシとする。 |
| **5** | **スコープ管理** | **どこまでやるのか(Boundary)**。要件漏れとスコープクリープを物理的に防ぐ。 |
| **6** | **スケジュール管理** | **どう進捗を追うのか**。WBSや報告頻度を定め、遅延の早期検知を可能にする。 |
| **7** | **コスト管理** | **いくらかかるのか(Money)**。予算内での完遂を保証し、投資対効果を守る。 |
| **8** | **品質管理** | **どのレベルで作るのか(Quality)**。テスト基準を定め、手戻りを最小化する。 |
| **9** | **資源管理** | **何を使ってやるのか(Resource)**。人・物・ライセンスの枯渇を防ぐ。 |
| **10** | **コミュニケーション管理** | **誰に何を伝えるか**。情報の偏りをなくし、チームの連動性を最大化する。 |
| **11** | **リスク管理** | **何が起きるか(What-if)**。ACE等を用い、不測の事態への備えを公式化する。 |
| **12** | **調達管理** | **外部をどう使うか**。ベンダー管理と契約遵守を徹底する。 |
| **13** | **統合管理** | **どう全体を束ねるか**。変更管理プロセスを定義し、一貫性を維持する。 |

## 2. マエストロの適用プロトコル
- **Initial Phase**: 項目1-5を重点的に固め、ステークホルダーの合意を取り付ける。
- **Execution Phase**: 項目6-11を運用し、`PERFORMANCE_DASHBOARD` や `ACE` と連動させる。
- **Close Phase**: 項目13に基づき、最終的な成果物と教訓（LEGEND）を整理する。
