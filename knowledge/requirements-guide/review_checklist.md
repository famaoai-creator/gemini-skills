# 要件定義ドキュメント・レビューチェックリスト

## 1. ビジネス要求
- [ ] **ステークホルダ**: 経営層、外部システム、保守運用が含まれているか？
- [ ] **目的の明確化**: 「なぜ(Why)」が経営目標（KPI）と紐付いているか？
- [ ] **スコープ**: Out-Scope（対象外）が明記されているか？

## 2. 業務プロセス
- [ ] **階層化**: ビジネスレベルとシステム利用レベルが分離されているか？
- [ ] **例外処理**: エラー、中断、キャンセルなどが網羅されているか？
- [ ] **5W2H**: フロー上の条件が明確か？

## 3. データ構造
- [ ] **概念データモデル**: 「モノ」と「コト」がER図で表現されているか？
- [ ] **整合性(CRUD)**: すべてのエンティティにCとRの機能があるか？
- [ ] **用語の統一**: シノニム（異名同義語）やホモニム（同名異義語）がないか？

## 4. インターフェース
- [ ] **UI標準**: 共通ルールが定義されているか？
- [ ] **画面遷移**: 行き止まりの画面はないか？
- [ ] **利用シーン**: 習熟者向けか初心者向けか明確か？

## 5. 非機能要求
- [ ] **数値化**: 秒数や稼働率が定義されているか？
- [ ] **網羅性**: 6項目（可用性、性能、拡張性、運用、移行、セキュリティ）が検討されているか？
- [ ] **トレードオフ**: 優先順位がついているか？
