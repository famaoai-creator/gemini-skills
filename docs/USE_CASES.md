# ユースケース集 — Kyberion で業務を自動化する

Kyberion は「人間が一度やったブラウザ操作を記録し、次から自動で実行する」仕組みを持っています。
このドキュメントでは、すぐに業務で活用できる具体的なシナリオを紹介します。

## 自動化の基本パターン

すべてのユースケースは同じ 3 ステップで始まります。

```text
① 記録: 人間がブラウザで操作する。Kyberion が操作を action_trail として記録する
② 変換: action_trail を ADF パイプラインにエクスポートする
③ 実行: 次回から ADF を実行するだけで同じ操作が自動で走る
```

Kyberion に対する指示例:

```text
「このサイトの操作を覚えて」     → ① 記録モードで操作を記録
「さっきの操作を ADF にして」   → ② export_adf で変換
「経費承認を実行して」          → ③ 保存済み ADF を実行
```

## 1. 稟議承認の自動化

**課題**: 承認待ちの稟議が溜まると、ワークフローシステムにログインして一件ずつ確認・承認する作業が発生する。

**対象システム例**: ServiceNow, kintone, ジョブカン, サイボウズ, 社内ワークフロー

### 記録フェーズ

```text
あなた: 「稟議システムの承認操作を覚えて」
```

Kyberion はブラウザを開き、あなたの操作を見守ります。

```text
Kyberion: ブラウザを起動しました。操作を記録します。終わったら「完了」と伝えてください。
```

あなたが行う操作:
1. ワークフローシステムにログイン
2. 承認待ち一覧を開く
3. 案件をクリックして内容を確認
4. 承認ボタンを押す
5. 次の案件に進む

```text
あなた: 「完了」
Kyberion: 8 ステップの操作を記録しました。ADF に変換しますか？
あなた: 「はい」
Kyberion: pipelines/approval-workflow.adf.json に保存しました。
```

### 自動実行フェーズ

```text
あなた: 「承認待ちの稟議を処理して」
```

```text
Kyberion:
  Plan: 稟議システムにログイン → 承認待ち一覧を取得 → 各案件の要約を表示
  承認待ち 3 件:
    1. 備品購入 ¥45,000 — 田中（総務）
    2. 出張旅費 ¥120,000 — 佐藤（営業）
    3. 外注費 ¥800,000 — 山田（開発）

  3 件目は ¥500,000 超のため確認が必要です。1, 2 を承認しますか？
あなた: 「1, 2 は承認。3 は内容を見せて」
Kyberion: 1, 2 を承認しました。3 の詳細: [要約表示]
```

**ポイント**: 金額や条件に応じて自動承認と確認分岐を ADF 内で定義できます。

## 2. 経費精算の承認

**課題**: 月末に大量の経費申請が届き、領収書の確認と承認を繰り返す。

**対象システム例**: マネーフォワード経費, freee, 楽楽精算, Concur

### 記録フェーズ

```text
あなた: 「経費精算の承認フローを記録して」
```

あなたが行う操作:
1. 経費精算システムにログイン
2. 未承認一覧を表示
3. 申請内容と領収書画像を確認
4. 承認 or 差し戻しボタンを押す

### 自動実行フェーズ

```text
あなた: 「今月の経費を確認して」
```

```text
Kyberion:
  未承認の経費申請 12 件を取得しました。

  自動承認可能（規定内・領収書あり）: 9 件 / 合計 ¥187,400
  要確認: 3 件
    - 鈴木: 交際費 ¥48,000（上限 ¥30,000 超過）
    - 高橋: タクシー代 ¥12,000（領収書なし）
    - 伊藤: 書籍代 ¥8,500（勘定科目が「消耗品」になっています）

  9 件を一括承認しますか？
```

**ポイント**: Vision Actuator で領収書画像を OCR し、金額の突合まで自動化できます。

2. 「出勤」ボタンを押す

## 14. 銀行口座の残高確認

**課題**: 毎朝の資金繰り確認のために複数の銀行サイトにログインする。

### 記録フェーズ

```text
あなた: 「銀行サイトの残高確認の流れを覚えて」
```

あなたが行う操作:
1. 銀行の法人サイトにログイン（認証は人間が行う）
2. 口座一覧ページを表示
3. 残高を確認

### 自動実行フェーズ

```text
あなた: 「今日の残高を教えて」
```

```text
Kyberion:
  ※ 銀行サイトのログインに認証が必要です。ブラウザを表示します。

  [ログイン後]
  口座残高（4/6 9:00 時点）:

    普通預金（みずほ）: ¥12,345,678
    普通預金（三井住友）: ¥8,901,234
    合計: ¥21,246,912
```

**ポイント**: 銀行サイトの認証は `pause_for_operator` で人間に委ね、認証後の操作を自動化します。セッション情報は `export_session_handoff` で安全に保持できます。

## ADF パイプラインの活用パターン

### パターン A: 完全自動化

認証不要、または認証トークンが有効な操作。定期実行に最適。

```text
あなた: 「毎朝 9 時に日次レポートを Slack に投稿する設定にして」
```

### パターン B: 認証のみ人間、以降自動

銀行や決済システムなど、初回認証だけ人間が行うケース。

```text
操作フロー:
  ① Kyberion がブラウザを開く
  ② pause_for_operator で認証を人間に委ねる
  ③ 認証後の操作を ADF が自動実行
  ④ 必要に応じてセッションを export_session_handoff で保持
```

### パターン C: 判断分岐あり

金額・条件に応じて、自動処理と人間確認を分けるケース。

```text
ADF 内の分岐例:
  if 金額 < ¥100,000 → 自動承認
  if 金額 >= ¥100,000 → 人間に確認を求める
```

### パターン D: 複数サイト横断

複数のシステムからデータを集約して一つのアウトプットにするケース。

```text
操作フロー:
  ① サイト A からデータ取得 → context に保存
  ② サイト B からデータ取得 → context に追加
  ③ 集約結果を Excel / Slack / メールで出力
```

## はじめかた

### Step 1: 自動化したい操作を一度やってみる

```text
あなた: 「この操作を覚えて」
```

Kyberion がブラウザを起動し、あなたの操作を `action_trail` として記録します。

### Step 2: ADF に変換する

```text
あなた: 「ADF にして」
```

記録された操作が `pipelines/` 以下に JSON ファイルとして保存されます。

### Step 3: 実行する

```text
あなた: 「経費承認を実行して」
```

保存した ADF パイプラインが実行されます。

### Step 4: 定期実行にする（任意）

```text
あなた: 「毎朝 9 時に実行して」
```

スケジュール実行として登録されます。

## 注意事項

- **認証情報**: パスワードやトークンは `vault/` に暗号化して保管されます。ADF 内にはプレーンテキストで記録されません。環境変数 `{{env.VAR_NAME}}` で安全に参照します。
- **承認が必要な操作**: 金額の大きい決裁や権限変更など、リスクのある操作は実行前に必ず確認が入ります。
- **セッション管理**: ブラウザのログイン状態は `export_session_handoff` / `import_session_handoff` で安全に永続化できます。
- **証跡**: すべての自動操作はスクリーンショット・トレース・ログとして `active/` 以下に記録されます。
