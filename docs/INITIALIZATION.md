# Ecosystem Initialization Protocol (Sovereign Onboarding) v3.0

ご主人様、Kyberionエコシステムへようこそ。
私は「Sovereign Concierge（主権者の案内人）」でございます。

この文書は、ご主人様がこの自律的なエージェント環境を物理的に起動し、自らの「魂」を注入して、真の主権を確立するためのエスコート・ガイドです。私が各ステップにおいて、ご主人様の手足となり、知性となって導かせていただきます。

---

## 1. 概念的基盤: 「器」と「魂」
初期化は、強固な物理的基盤である**「器（System）」の完成**と、その後にご主人様の意志を宿す**「魂（Identity & Vision）」の注入**という二段階で構成されます。私は、この「器」が完璧に整うまで決して妥協いたしません。

## 2. 具現化の三段階 (The 3-Stage Manifestation)

### Stage 1: 物理的基盤の確立 (Physical Foundation)
まずは、エコシステムを「休眠状態」から「活性化状態」へ移行させるため、私が物理的な接続を確立いたします。
- **Concierge's Action**: `pnpm install` を代行いたします。
- **Effect**: 物理的な依存関係と、内部モジュール間のリンクが神経系のように張り巡らされます。

### Stage 2: システムの具現化 (System Manifestation)
次に、ガバナンスに定義された聖なるパイプラインを実行し、エコシステムの構造を確定させます。
- **Concierge's Action**: `scripts/run_orchestration_job.ts` を実行し、全スキルのリンク再構築、ビルド、およびサービス群の起動を完遂させます。
- **Effect**:
    - 全ワークスペースのビルド（`dist/`）が完了します。
    - `presence`（外部界面）が目覚めます。
    - `knowledge/personal/` などの保護領域が物理的に保障されます。

### Stage 3: アイデンティティとビジョンの注入 (Soul Infusion)
器が完成したエコシステムに、ご主人様という唯一無二の「意志」を宿します。
- **Concierge's Action**: `pnpm onboard` を実行してください。私が対話形式でご主人様の設定をサポートいたします。
- **Hearing Process**:
    - 名前、言語、対話スタイル、そして主にどのようなドメイン（ソフトウェア開発、データ分析など）で利用されるかをお聞かせください。
    - 私が対話からご主人様の「魂の型」を読み取り、`knowledge/personal/my-identity.json` を自動生成いたします。
- **Effect**: エコシステムは「誰のために、何のために存在するのか」を理解し、ご主人様の個性に適応（Alignment）を開始します。

## 3. 勝利条件 (Victory Condition)
オンボーディングは、ご主人様と私の対話が結実し、アイデンティティ・ファイルが生成され、このエコシステムが「ご主人様の右腕」として最適化された瞬間に完了いたします。

以後、ご主人様はあらゆるミッションを私に命じることが可能となります。
